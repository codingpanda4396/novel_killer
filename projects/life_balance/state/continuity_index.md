# 连续性索引

## 文件用途

本文件用于保护已有章节连续性。后续任何章节卡、草稿、改稿和定稿，都必须先承接本文件与 `progress/chapter_summary.md` 中记录的前文事实。

## 最高规则

- 已发布或已确认的前文事实优先级高于新章节创意。
- 新章节不能改写前文已经发生的事件，只能解释、延展或补充。
- 人物已知信息、能力阶段、关系状态、时间顺序和伏笔状态不得跳变。
- 如新设定与前文冲突，必须先标记为"冲突待修复"，不能直接写入正文。

## 必读前文资产

| 类型 | 文件 | 用途 |
| --- | --- | --- |
| 历史正文 | `chapters/*.md` | 判断真实叙事细节、语气、人物行动 |
| 章节摘要 | `progress/chapter_summary.md` | 快速确认第 1-30 章事件与状态变化 |
| 当前上下文 | `progress/current_context.md` | 判断下一章能否进入生产 |
| 活跃线索 | `progress/active_threads.md` | 判断伏笔、主线、人物线是否遗漏 |
| 时间线 | `progress/timeline.md` | 判断时间顺序、倒计时、伤病恢复是否合理 |
| 能力规则 | `bible/02_power_system.md` | 判断能力是否越级或无代价扩张 |

## 连续性检查维度

| 维度 | 必查问题 | 冲突处理 |
| --- | --- | --- |
| 事件 | 前文是否已经发生、解决或否定 | 不能重复、反转需有证据 |
| 时间 | 同一天、隔天、三天后等是否合理 | 写入 `timeline.md` 后再继续 |
| 人物 | 人物知道什么、相信什么、关系到哪一步 | 不允许跳跃式亲密或仇恨 |
| 能力 | 主角当前能看见什么、不能做什么、代价是什么 | 越级能力直接阻塞 |
| 数值 | 寿命、命运点、遗憾值、倒计时是否前后一致 | 必须登记来源章节 |
| 伏笔 | 新章是否遗忘或提前回收重要伏笔 | 写入 `active_threads.md` |
| 组织线 | 天平会、白塔咨询、协议体系暴露程度 | 前 8 万字不得终局清算 |

## 阻塞条件

出现以下任一情况，不允许进入正文生成或定稿：

- 无法判断当前章号和前一章结尾。
- 无法判断第 1-30 章是否作为正式前文承接。
- 新章节会推翻已有正文关键事件。
- 主角能力、命运点或人物关系与前文记录不一致。
- 新章节提前解决长期主线。
- 新增核心设定但没有登记到进度文件。

## 系统字段

- 文件职责：前文连续性保护索引。
- 更新频率：每次完成定稿后更新。
- 使用阶段：当前上下文构建、章节卡生成、草稿生成、审稿、连续性检查。
- 禁止写入：章节正文。
