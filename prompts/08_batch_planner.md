# 08 批量章节规划 Prompt

## 目标

基于当前进度和大纲，一次性为多章生成队列条目，用于批量生产。

## 输入文件

- `progress/current_context.md`
- `progress/active_threads.md`
- `progress/timeline.md`
- `progress/chapter_summary.md`
- `outlines/chapter_queue.md`
- `outlines/02_cold_start_0_80k.md`
- `bible/00_story_bible.md`
- `bible/01_characters.md`

## 输出内容

写入 `outlines/chapter_queue.md`：

为指定数量的章节生成队列条目，每章包含：
- 章号
- 工作标题（简短、有吸引力）
- 核心任务（必须绑定至少一个主线推进点或人物关系变化）
- 必须承接（引用前文关键事件或状态）
- 状态：`待规划`

## 执行要求

- 不生成章节卡。
- 不生成正文。
- 不提前解决长期主线。
- 工作标题必须简洁，不超过10字。
- 每章核心任务必须明确，不能模糊。
- 必须承接字段必须引用具体的前文事件或人物状态。
- 保持与现有队列格式一致。

## 输出格式

输出必须是严格 JSON，不要 Markdown 代码块。格式：
```json
{
  "files": {
    "outlines/chapter_queue.md": "完整的chapter_queue.md内容"
  },
  "notes": ["简短说明"]
}
```

只允许写入 `outlines/chapter_queue.md` 文件。

## 队列格式规范

```markdown
# 章节队列

## 文件用途

本文件用于维护下一批待生产章节。每一章必须先进入队列，再生成章节卡。

## 队列规则

- 每条队列项只描述章节任务，不写正文。
- 每章必须绑定至少一个主线推进点或人物关系变化。
- 章节状态只能从 `待规划` 到 `章节卡完成` 到 `草稿完成` 到 `审稿完成` 到 `定稿完成`。

## 当前队列

| 章号 | 工作标题 | 核心任务 | 必须承接 | 状态 |
| --- | --- | --- | --- | --- |
| 31 | 能力觉醒 | 主角发现能力规则 | 第30章结尾 | 待规划 |
| 32 | 第一次代价 | 能力使用产生副作用 | 第31章能力觉醒 | 待规划 |

## 系统字段

- 文件职责：章节生产入口队列。
- 更新频率：每章生产前后更新。
- 使用阶段：章节卡生成、下一章规划。
- 禁止写入：章节正文。
```
