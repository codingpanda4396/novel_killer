# Novel0 项目总结报告

## 📚 项目概述

**项目名称**：Novel0 - 长篇连载小说自动生成系统

**小说标题**：《我能看见人生余额：开局救下白月光》

**项目类型**：AI辅助的长篇小说创作系统

**目标平台**：番茄小说（商业连载）

**当前状态**：第一卷完成（50章，约12万字）

---

## 🎯 项目目标

### 核心目标
1. **商业化导向**：以番茄小说流量增长为核心目标
2. **工业化生产**：建立可复制的长篇小说生产流水线
3. **质量保证**：确保逻辑连贯性、商业性、可读性

### 技术目标
1. 实现一键批量生成章节
2. 自动化质量检查和连续性保护
3. 断点续传和进度追踪
4. 完整的故事约束和世界观管理

## 🏗️ 系统架构

### 1. 目录结构
```
novel0/
├── bible/                  # 故事约束（世界观、人物、能力、文风）
│   ├── 00_story_bible.md
│   ├── 01_characters.md
│   └── 02_power_system.md
├── outlines/               # 大纲、章节队列、修复方案
│   ├── 04_VOLUME_OUTLINE.md
│   ├── chapter_queue.md
│   └── volume_01_repair_plan.md
├── chapters/               # 正文章节（50章已完成）
│   ├── 01-50.md           # 正式章节
│   ├── drafts/            # 草稿
│   ├── scene_cards/       # 章节卡
│   ├── records/           # 审稿记录、连续性检查
│   └── final/             # 定稿
├── progress/              # 进度追踪
│   ├── current_context.md
│   ├── chapter_summary.md
│   ├── continuity_index.md
│   ├── open_threads.md
│   └── character_state.md
├── prompts/               # LLM提示词模板（9个阶段）
├── scripts/
│   └── novel_workflow.py  # 核心自动化脚本
├── config/
│   └── workflow.local.json # 配置文件
└── Makefile               # 命令入口
```

### 2. 生产流水线

每章自动经过7个阶段：
1. **build_context** - 更新当前上下文
2. **scene_card** - 生成章节卡（前文承接、本章任务、禁止事项）
3. **draft** - 生成草稿（2000-2500字）
4. **review** - 审稿（商业性、人物、节奏、设定检查）
5. **continuity_check** - 连续性检查（防止矛盾）
6. **update_record** - 更新进度文件
7. **next_planner** - 规划下一章

### 3. 核心功能

#### 一键批量生成
```bash
make run-batch START=31 COUNT=10 CHECK=1
```

#### 断点续传
```bash
make run-batch START=31 COUNT=10 RESUME=1
```

#### 质量检查
- 自动连续性检查
- 商业性评分
- 人物一致性验证
- 设定冲突检测

## 📖 故事内容

### 核心设定

**主角**：陈默，大学生，获得"人生余额可视"能力

**能力体系**：
- 可以看见他人头顶的寿命、财运、情缘倒计时
- 干预命运需要付出代价（消耗自身寿命）
- 通过救赎他人获得"命运点"，解锁更高能力
- 能力成长路径：规则感知（7点）→ 规则理解（10点）

**核心冲突**：
- 个人层面：救人vs付出代价的道德困境
- 组织层面：对抗"天平会"（买卖命运的神秘组织）
- 规则层面：进入命运维度，重写核心系统

**情感线**：
- 主线：陈默与林见鹿（被救的女主）
- 副线：父子线（父亲曾是天平会锚点）
- 传承线：教导新能力者，建立"余额守护者"

### 故事结构（第一卷）

**第1-10章**：能力觉醒
- 第1章：救下林见鹿，发现能力
- 第2-3章：理解能力，救室友周野
- 第4-10章：单元救赎案例，积累经验

**第11-20章**：真相揭露
- 第11-13章：发现天平会存在
- 第14-15章：得知父亲是锚点
- 第16-20章：初次对抗天平会

**第21-30章**：第一次高潮
- 第21-25章：潜入数据中心
- 第26-30章：摧毁数据中心，父亲获救
- 第30章：发现核心系统仍在，寿命只剩30天

**第31-40章**：父亲线+反制
- 第31-35章：发现父亲寿命被锁定，破解协议痕迹
- 第36-40章：植入反制程序，对决顾沉舟

**第41-50章**：命运维度+大结局
- 第41-42章：救赎案例，达到10命运点
- 第43-44章：进入命运维度，父亲牺牲，重写系统
- 第45-50章：新世界，建立"余额守护者"，圆满结局

## 🎨 创作特色

### 1. 能力体系完整
- 有边界：每天最多使用3次，有代价
- 有成长：命运点可视化，能力分级明确
- 有限制：看不见被保护的部分，权限不足

### 2. 商业性优化
- **节奏控制**：每章2000-2500字，紧凑不拖沓
- **爽点密度**：每3-5章一个高潮
- **悬念设置**：每章结尾强钩子
- **冲突升级**：个人→组织→规则，层层递进

### 3. 情感饱满
- **父子线**：从隔阂到理解到牺牲
- **爱情线**：从相救到相知到相守
- **师徒线**：传承主题，教导新能力者

### 4. 主题深刻
- **核心主题**：人生不能被定价
- **副主题**：自由、选择、责任、传承
- **社会意义**：反对命运交易，倡导自由选择

**第2章**：
- 林见鹿先情绪失控，第二天才冷静
- 对话更自然，减少说教感
- 增加动作描写和情绪变化

**第3章**：
- 增加陈默管周野前的内心挣扎（500字）
- 增加寿命异常线索
- 强化"嘴硬心软"性格标签

#### 3. 第31-50章生成（全新内容）
**特色**：
- 3个完整救赎案例（张雨弟弟、李明、刘建国/王强）

## 🚀 技术创新

### 1. 自动化流水线
- **一键生成**：指定起始章节和数量，自动连续生产
- **断点续传**：中断后自动从上次位置继续
- **质量检查**：每章生成后自动运行连续性检查
- **进度追踪**：实时更新进度文件

### 2. 连续性保护
- **前文事实锁定**：禁止推翻已确立的设定
- **人物状态追踪**：实时更新角色状态
- **伏笔管理**：记录未回收伏笔
- **时间线管理**：防止时间线混乱

### 3. 模块化设计
- **Bible系统**：故事约束、人物、能力、文风分离
- **Progress系统**：当前上下文、章节总结、伏笔清单分离
- **Prompts系统**：9个阶段的提示词模板化
- **Config系统**：API配置、模型选择可配置

### 4. 混合生产模式
- **AI生成**：使用LLM生成草稿
- **人工优化**：关键章节人工精修
- **混合方案**：AI框架+人工润色
