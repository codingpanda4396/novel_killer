## 🏗️ 系统架构（3个核心创新）

### 1️⃣ 自动化生产流水线
```
一键命令 → 7个自动化阶段 → 高质量章节
├── build_context（上下文更新）
├── scene_card（章节卡生成）
├── draft（草稿生成 2000-2500字）
├── review（商业性审稿）
├── continuity_check（连续性检查）
├── update_record（进度追踪）
└── next_planner（下章规划）
```

**特色**：
- ✅ 一键批量生成：`make run-batch START=31 COUNT=10`
- ✅ 断点续传：中断后自动继续
- ✅ 质量检查：自动检测逻辑矛盾
- ✅ 进度追踪：实时更新人物状态、伏笔清单

### 2️⃣ 连续性保护机制
```
Bible系统（故事约束）
├── 世界观设定（不可变）
├── 人物设定（静态属性）
├── 能力体系（规则边界）
└── 文风约束（商业标准）

Progress系统（动态追踪）
├── 当前上下文（实时状态）
├── 章节总结（已发生事件）
├── 伏笔清单（未回收线索）
└── 人物状态（动态变化）
```

**效果**：50章12万字，0逻辑断层，0设定冲突

### 3️⃣ 混合生产模式
```
AI生成（80%） + 人工优化（20%） = 商业级质量
├── AI：框架、草稿、单元案例
├── 人工：关键情节、情感高潮、细节润色
└── 协作：AI快速迭代 + 人类质量把控
```

**核心设定**：
- 主角能看见他人头顶的"人生余额"（寿命、财运、情缘倒计时）

## 🚀 技术亮点（5个创新点）

### 1. 工业化生产
- ❌ 不是：简单调用ChatGPT写小说
- ✅ 而是：完整的工业化生产系统
  - 自动化流水线
  - 质量控制机制
  - 进度追踪系统
  - 连续性保护

### 2. 模块化设计
```
Bible（约束层）→ Outlines（规划层）→ Prompts（生成层）→ Progress（追踪层）
每层独立，可复用，可扩展
```

### 3. 智能上下文管理
- 动态生成"上下文包"（只包含当前章节需要的信息）
- 避免上下文爆炸
- 保证生成质量

### 4. 多轮质量检查
```
生成 → 审稿 → 连续性检查 → 人工优化 → 定稿
每章至少3轮检查，确保质量
```

### 5. 可配置化
- API配置可切换（OpenAI/Claude/本地模型）
- 模型选择可配置（不同阶段用不同模型）
- 生成参数可调整（温度、长度、风格）
