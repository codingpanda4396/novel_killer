# 05 章节生产规范

## 章节基础参数

每章正文要求：

- 字数：2000—2500字。
- 章节标题：必须有悬念或爽点。
- 开头：200字内进入冲突。
- 中段：推进一个主要事件。
- 结尾：必须有钩子。
- 人物：每章最多重点写2—3个角色。
- 设定：每章最多新增1个设定点。
- 信息：每章必须有一个新信息或状态变化。

## 单章标准结构

推荐结构：

```text
1. 开头钩子：异常、危险、冲突、倒计时、情绪爆点。
2. 目标明确：主角本章要解决什么。
3. 阻碍出现：能力限制、人物阻拦、信息误导、时间压力。
4. 主角行动：陈默用观察、推理、能力或现实手段破局。
5. 反转：事情不是表面看到的样子。
6. 阶段结果：救人、失败、获得线索、关系推进。
7. 章末钩子：新倒计时、新人物、新协议、新危机。
```

## 章节输入格式

每次让 Codex 写章节前，必须提供：

```markdown
# 当前章节任务

## 章节编号
第 X 章

## 章节标题
标题

## 所属卷
第 X 卷：卷名

## 本章核心任务
一句话说明本章必须完成什么。

## 上章结尾
简述上章留下的钩子。

## 本章开头承接
本章从哪里开始。

## 本章必须出现
- 场景
- 人物
- 关键道具/数值
- 冲突
- 新信息

## 本章禁止出现
- 不准提前透露的信息
- 不准解决的问题
- 不准出现的人物或设定

## 本章结尾钩子
明确本章最后要钩住什么。
```

## 章节输出格式

Codex 产出章节时，建议使用“两段式输出”。

### A. 正文模式

只输出读者能看到的章节正文：

```markdown
# 第X章 标题

正文……
```

### B. 控制模式

正文后附加内部检查记录。这个部分不发布，只用于作者管理：

```markdown
---

## 作者控制记录

### 本章完成的剧情任务
-

### 本章新增设定
-

### 本章人物关系变化
-

### 本章留下的钩子
-

### 需要写入连续性记录的内容
-
```

## 开头钩子类型

优先使用以下开头：

1. 倒计时开头：`林见鹿头顶的数字，只剩下三分钟。`
2. 异常数值开头：`陈默第一次看见，一个人的财运余额是负数。`
3. 反常行为开头：`那个外卖员站在马路中央，像是在等一场必然发生的车祸。`
4. 危险结果开头：`电梯门合上的下一秒，整栋商场的灯同时灭了。`
5. 关系冲突开头：`林见鹿说：“陈默，你是不是早就知道我不该活着？”`

## 结尾钩子类型

每章必须从以下类型中选一个：

1. 新倒计时出现。
2. 新人物出场。
3. 关键数值反常。
4. 男主付出代价。
5. 女主发现异常。
6. 反派露出痕迹。
7. 主角判断被推翻。
8. 已解决事件出现更大问题。
9. 父亲线索浮现。
10. 天平会协议出现。

示例结尾：

```text
陈默刚想松一口气，却看见自己手腕上浮现出一行新的蓝色小字。

寿命余额：30天。
```

## 单章节奏比例

一章2200字左右时，可按：

- 300字：开头钩子与承接。
- 500字：场景推进与人物冲突。
- 600字：调查、行动或对话博弈。
- 500字：反转与阶段结果。
- 300字：情绪落点与章末钩子。

## 对话比例

每章对话建议占30%—45%。不要连续大段说明。用对话暴露信息时，必须带冲突。

错误：

```text
林见鹿解释了命运余额的原理。
```

正确：

```text
林见鹿盯着他：“你每救一个人，自己的寿命就会减少，对吗？”
陈默沉默。
她的声音低了下去：“那你刚才救我，少了多少？”
```

## 爽点类型

每3—5章必须出现至少一个爽点：

- 救人成功
- 反杀骗局
- 当众打脸
- 提前预判
- 女主改变态度
- 赚钱到账
- 反派吃瘪
- 能力升级
- 新线索爆出
- 身份反转

## 不可用章节

以下章节视为失败：

- 纯铺设定，无事件。
- 纯谈恋爱，无主线。
- 纯讲道理，无剧情。
- 主角没行动，只听别人说。
- 女主只被动等待。
- 章末没有任何悬念。
- 一章里新增超过三个重大设定。
- 大量解释世界观导致节奏停滞。
