# 当前上下文

## 文件用途

本文件是每次生产章节前必须读取的即时上下文。它只记录“现在写下一章必须知道什么”。

## 当前生产状态

- 当前目标：前 8 万字冷启动系统初始化。
- 当前章号：待确认。
- 当前卷：待确认。
- 当前阶段：禁止直接生成正文，先完成冷启动规划。
- 下一步入口：运行 `prompts/00_build_current_context.md`。

## 必须承接

- 已有第 1-30 章历史正文资产。
- `progress/system_audit.md` 指出第 21-30 章存在终局化风险。
- 后续需要判断历史正文是正式保留、局部修复，还是作为试生产样章。
- 后续生成不得破坏第 1-30 章已经建立的人物、能力、事件、时间和伏笔连续性。
- 进入任何章节卡或正文生成前，必须先读取 `progress/continuity_index.md` 和 `progress/chapter_summary.md`。

## 当前禁止

- 禁止生成具体章节正文。
- 禁止把现有“第一卷完结”直接作为长期连载状态继续推进。
- 禁止跳过章节卡。
- 禁止在连续性状态未确认时生成新章节。
- 禁止为新剧情便利修改前文事实。

## 系统字段

- 文件职责：章节生产前的即时上下文。
- 更新频率：每次生产前和每章完成后更新。
- 使用阶段：所有 prompt 的首要输入。
- 禁止写入：完整章节正文。
